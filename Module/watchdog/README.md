# 看门狗使用文档  
几个步骤：绑定行为->注册->定时poll->喂狗  

poll就是用一个任务去定期检查  

看门狗可以设置为三种模式：  
- ONE_SHOT: 触发一次，需要手动恢复  
- AUTO_REARM: 恢复后自动继续监控  
- LATCHED: 触发后，必须disarm()->arm()

喂狗，比如在电机类中，在onRx中去喂狗  

参数设置，有timeout超时时间 和 debounce容忍次数  


可以绑定看门狗超时行为  


目前我的程序设计是：
1. 电机类内持有看门成成员，onRx中喂狗，但是还无绑定离线行为，电机离线cmd赋值会直接砍0，但是这里还没有解决pid控制器积分累加的问题    
2. watchdog_task注册所有需要看门狗的设备（extern），watchdog_task中绑定电机离线行为（清空pid输出），watchdog_task中poll检查所有挂载的看门狗设备  

DJI电机我的timeout设置了50ms，debounce设置了2  
可以试一下（把电机can拔了模拟电机离线，然后检查对应pid的输出值有无清0且电机看门狗是否处于TRIGGERED；然后插回can线检查电机看门狗是否恢复ARMED）  
当然这个阈值也不一定就合适，可以多试试  
当然后续想暴露到调试接口也很容易，直接给一个task扫描所有电机的看门狗成员，看其是否offline，如果offline就亮个对应指示灯就ok了  

当然这个看门狗也可以嵌入到别的模块中，比如遥控器、传感器等等... 具体嵌入方式参考电机类
